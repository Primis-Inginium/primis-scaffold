# 🤖 GEMINI.md - Primis Scaffold Context

This file serves as the foundational context for Gemini CLI interactions within the **Primis Scaffold** project. It outlines the project's purpose, architecture, and development standards.

## 🌟 Project Overview
**Primis Scaffold** is a standardized Next.js template for **Primis-Inginium** applications. It is designed for high AI efficiency and follows a strict modular architecture.

- **Main Technologies:**
  - **Framework:** [Next.js 14](https://nextjs.org/) (App Router)
  - **Language:** TypeScript
  - **Styling:** Tailwind CSS
  - **Icons:** Lucide-React
  - **Testing:** Playwright

## 🏗 Modular Architecture
The project adheres to a strict **Core-Feature** pattern to ensure scalability and maintainability.

- **`@core/*` (`src/core/`):** Generic, reusable primitives (components, API utilities, state managers).
- **`@features/*` (`src/features/`):** Domain-specific modules (e.g., `auth`, `user-profile`). Each subdirectory should be a self-contained feature.
- **`docs/discovery/`:** Stores project discovery documents, including transcripts, structured requirements, and client approvals.

## 🚀 Building and Running
The following commands are standard for this project:

- **Development:** `npm run dev` (Starts the Next.js development server)
- **Build:** `npm run build` (Builds the application for production)
- **Production Start:** `npm run start` (Starts the production server)
- **Linting:** `npm run lint` (Runs ESLint checks)
- **Testing:** `npx playwright test` (Runs end-to-end tests)

## 🛡️ Development Conventions & Governance
To maintain high standards and AI compatibility, follow these guidelines:

1.  **AI Governance:** Inherits all organization-level governance from the Primis-Inginium repository.
2.  **Modular Integrity:** Ensure generic logic is placed in `@core` and domain-specific logic is encapsulated within its respective feature in `@features`.
3.  **Audit Logs:** All Pull Requests must include a mandatory audit log of the changes made.
4.  **Discovery-First:** Technical implementations should be traced back to the requirements in `docs/discovery/`.
5.  **Strict Typing:** Maintain strict TypeScript compliance as configured in `tsconfig.json`.
